# 🚀 就活状況管理アプリ（WebApplication_kadai）

就職活動における企業情報の一覧表示、お気に入り登録、および選考ステータスの管理を行うWebアプリケーションです。  
**「散らばりがちな企業情報を一元管理し、現在の選考フェーズを可視化する」**ことを目的として制作しました。

---

## 🔗 デプロイURL

[https://web-application-kadai-wheat.vercel.app](https://web-application-kadai-wheat.vercel.app)

---

## ✨ 主な機能

- **企業一覧表示**: microCMSから取得した最新の企業情報をカード形式で一覧表示。
- **お気に入り機能**: 気になる企業を「お気に入り」として保存し、後から素早く確認可能。
- **選考ステータス管理**: 各企業の選考状況（書類選考、面接、内定など）を個別に管理・更新。
- **レスポンシブ対応**: PC・スマートフォンの両方で快適に操作可能なデザイン。

---

## 📄 ページ構成

| ページ           | パス        | 説明                                                   |
| :--------------- | :---------- | :----------------------------------------------------- |
| **トップページ** | `/`         | 全企業の一覧を表示。タグによるフィルタリングが可能。   |
| **お気に入り**   | `/favorite` | 「お気に入り」に登録した企業のみを抽出して表示。       |
| **選考状況**     | `/status`   | エントリー済み企業の進捗（フェーズ）を一目で確認可能。 |

## 🛠 技術的な取り組み（microCMS連携）
データの永続化にはヘッドレスCMSである **microCMS** を使用しています。

* **データ取得**: `microcms-js-sdk` を使用。サーバーコンポーネントから直接APIを叩くことで、高速なレンダリングを実現。
* **型安全**: TypeScriptを用いてAPIレスポンスの型を定義し、データ取得から表示まで堅牢な開発を徹底。
* **環境変数の利用**: APIキーやサービスドメインは `process.env` を介して管理し、セキュリティを確保。

---

## 💡 工夫した点
* **ユーザー体験（UX）の向上**: `CSS Modules` を活用し、スタイルをコンポーネントごとにカプセル化（Scoped CSS）。デザインの崩れを防ぎつつ、保守性を高めました。
* **パフォーマンス最適化**: `Server Components` を積極的に利用。ブラウザへ送るJavaScriptを最小限に抑え、ページ読み込みを高速化しました。
* **直感的なUI**: ヘッダーから主要機能へ迷わずアクセスできるよう、シンプルかつ明快なナビゲーションを設計しました。

---

## ⚡ 苦労した点
### 1. 選考状況に応じた動的な表示制御
「選考状況ページ」にて、ステータスが「選択してください（初期値）」の企業を表示させないロジックの実装に苦労しました。条件分岐でこれを除外することで、ユーザーが必要な情報（エントリー済み企業）のみが表示されるよう工夫しました。

### 2. タグによるフィルタリング機能
企業一覧ページにて、特定のタグをクリックした際に該当企業を抽出する機能を実装しました。microCMSの複雑なデータ構造を TypeScript で正しく定義し、JavaScript の `filter` メソッドを組み合わせて、ミスなく表示を更新させるロジックを構築しました。
